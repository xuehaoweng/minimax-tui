Agent 能做什么

把一次自然语言需求拆成可执行步骤：读取项目、修改文件、运行命令、连接外部能力，并在失败时自动回退或重试。

Think-Act-Observe-Repeat Read / Write / Execute / Connect Sub-agent 并行

1. 读懂项目

快速索引目录结构、读取关键文件、搜索调用链，形成“任务上下文”。

  • 定位入口与依赖关系
  • 识别约束文件（策略、hooks、memory）
  • 提取可复用实现路径

2. 修改与创建文件

按需求精确改动代码、文档、配置，保持最小变更集并可追溯。

  • 新建页面、组件、脚本
  • 局部修复 bug / 重构片段
  • 遵守项目已有风格和约束

3. 执行验证

运行构建、测试、静态检查，自动采集错误并反馈下一轮修复。

  • 构建失败定位
  • 测试回归验证
  • 命令输出摘要化

4. 连接外部能力

通过 Connect/MCP 对接外部系统，扩展工具边界而不破坏主流程。

  • Web 搜索 / 抓取
  • MCP tools list / call
  • Hook 前后置管控
交互理念： 用户给目标，Agent 负责落地。任务长时自动做上下文压缩与长期记忆沉淀，确保可持续执行。
可直接执行 需确认配置 失败可回滚/重试
